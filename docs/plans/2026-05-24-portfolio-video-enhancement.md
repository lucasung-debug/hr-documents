# 포트폴리오 데모 영상 강화 기획서 v2

> 파이프라인: `/socrates`(방향 탐색) → `/doubt`(전제 반증·재건) → `/council`(3인 리뷰 반영)
> 9종 풀세트 대신 **단일 강화 기획서**로 압축 (기존 프로젝트 강화 + 단순성 원칙)

---

## 1. Cogito (무너지지 않는 출발점)

> **"나는 HR 분야로 취업·이직하기 위해, 이 HR 온보딩 프로젝트를 'HR 실무를 깊이 이해한 사람이 만든 프로덕션급 제품'임이 데모 영상 한 편으로 즉시 전달되는 상태로 만든다."**

**코기토 3축:** ① 목적축 = HR 취업·이직(포트폴리오 완성도는 *수단*) ② 인상축 = "프로덕션급 + HR 실무 깊이"가 *보임* ③ 매개축 = 라이브 클릭이 아닌 **데모 영상**(포트폴리오 임베드 + 면접 시연).

---

## 2. Assumption Audit (소각/생존 요약)

핵심 소각 3개가 로드맵을 영상 중심으로 재편했다.
- **A2 소각**: 목적은 포트폴리오 완성도가 아니라 HR 취업·이직.
- **A5 소각/재정의**: 치명 갭은 "데모 부재"가 아니라 "영상에 담길 매끄러운 플로우·스토리".
- **A6 소각**: 평가자는 데모를 직접 클릭하지 않는다 → **데모 영상**이 평가 매개(마스터가 비개발자).

(전체 가정 감사는 council-report 및 doubt 대화 기록 참조)

## 4. Reconstruction (3단계 로드맵 — Council 반영)

### 🎬 1단계 — 촬영 가능 상태 (Foundation) `대폭 경량화됨`
- **D3 데모 모드** (온보딩 플로우):
  - 가짜 사원 픽스처 로그인 (Google Sheets `findEmployeeByNameAndPhone` 우회)
  - 전자서명(캔버스) → 서류 동의 → **완료**까지 외부 API 없이 구동
  - **PDF는 미리 생성한 서명 샘플 PDF 1종을 표시** (Sheets 재구현 회피). 생성 과정은 로딩 연출.
- **🔒 데모 모드 격리 (HIGH, Council)**: 환경변수 플래그만으로 부족 → 빌드타임 격리 또는 `demo: true` JWT 클레임 + 실데이터 미들웨어 차단. 프로덕션에서 절대 활성화 불가하게.
- 빌드/구동 막는 이슈만 최소 처리.
- **완료 기준**: `npm run dev` 직후 로그인~완료가 외부 API 없이 끊김 없이 구동 + 기존 121 테스트 통과.

### ✨ 2단계 — 영상에 비칠 화면 강화 (Polish & Domain)
- **D4** 랜딩/오프닝 (`app/page.tsx` redirect → 소개 페이지): 문제 30초 + 핵심 기능 + 데모 진입.
- **D5** UX 폴리싱 — **영상 임팩트 우선순위로 제한**(Council): 에러 토스트 > 대시보드 로딩 스켈레톤(1회) > (빈 상태·모바일은 영상에 안 나오면 생략).
- **D7** 화면 속 HR 디테일:
  - 서명 직전 **"서명 무결성 검증"** 고지 (전자서명법 법적 주장 금지). 필요 시 "법적 전자서명은 외부 서비스 연동 가능"으로 로드맵 언급.
  - 완료/관리자 화면에 서명 감사정보(시각·해시 앞 8자리) 표시. **화면 텍스트는 핵심 한 줄만 크게.**
- **D6** 관리자 대시보드 진행률 차트 (숫자 카드 → 시각화).
- **D8** 페이지 레벨 admin 가드(`app/admin/layout.tsx` role 체크 → employee redirect). 추가 시 **기존 admin 경로 테스트 영향 점검**(Council). 접근거부 사유 화면으로 격상.

### 🎥 3단계 — 영상화 (Video Production)
- **D1** 시나리오·대본 확정(5절).
- **D2** Playwright 자동 구동·녹화(`video: on`) → webm → 필요 시 mp4.
- **2컷 산출**(Council): 2:30 풀버전 + 60초 포트폴리오 임베드 컷.
- **🔒 녹화 전 PII/시크릿 체크리스트**(HIGH, Council) 통과 필수(6절).
